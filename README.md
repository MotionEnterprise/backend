# Django Project Setup Guide

## Project Structure & Purpose

### Root Level
- `backend` - Main project root directory containing all project files
- `venv/` - Python virtual environment for isolated package management
- `manage.py` - Django's command-line utility for administrative tasks
- `.env` - Environment variables file (secret keys, API keys, database credentials)
- `.env.example` - Template for environment variables with placeholder values
- `.gitignore` - Specifies files/folders Git should ignore (secrets, cache, etc.)
- `README.md` - Project documentation with setup instructions and overview
- `docker-compose.yml` - Docker configuration for containerized deployment (optional)

### config/
Django project configuration directory
- `settings.py` - Main Django settings (database, installed apps, middleware, etc.)
- `urls.py` - Root URL configuration routing requests to appropriate apps
- `wsgi.py` - WSGI configuration for production deployment
- `asgi.py` - ASGI configuration for async support
- `celery.py` - Celery configuration for background task processing

### apps/
Contains all Django applications (business logic separated by domain)

#### apps/accounts/
User management and subscription handling
- `models.py` - User, UserProfile, Subscription, Plan, Payment models
- `views.py` - API views/viewsets for user and subscription endpoints
- `serializers.py` - DRF serializers for JSON conversion of user/subscription data
- `services.py` - Business logic for user operations and subscription management
- `stripe_utils.py` - Stripe payment integration utilities
- `admin.py` - Django admin interface configuration
- `apps.py` - App configuration
- `tests/` - Unit and integration tests for accounts functionality

#### apps/library/
Content library management (prompts, workflows, industries)
- `models.py` - Prompt, Workflow, Industry, Template, Collection models
- `views.py` - API views for library content CRUD operations
- `serializers.py` - DRF serializers for library content
- `services.py` - Business logic for managing prompts, workflows, and industries
- `admin.py` - Django admin configuration for library content
- `apps.py` - App configuration
- `tests/` - Tests for library functionality

#### apps/generation/
AI generation engine and job queue management
- `models.py` - GenerationJob, JobQueue, GenerationHistory models
- `views.py` - API views for creating and managing generation jobs
- `serializers.py` - DRF serializers for generation job data
- `services.py` - Business logic for AI generation orchestration
- `tasks.py` - Celery tasks for asynchronous background job processing
- `admin.py` - Django admin for job monitoring
- `apps.py` - App configuration
- `tests/` - Tests for generation and job processing

#### apps/media/
Asset and file management (images, videos, documents)
- `models.py` - Asset, AssetMetadata, AssetTag models
- `views.py` - API views for asset upload/download and management
- `serializers.py` - DRF serializers for asset data
- `services.py` - Business logic for asset operations
- `storage.py` - S3 integration and custom storage backends
- `admin.py` - Django admin for asset management
- `apps.py` - App configuration
- `tests/` - Tests for media handling

### core/
Shared utilities used across all apps (not a Django app)
- `utils.py` - Common utility functions (formatters, helpers, validators)
- `constants.py` - Project-wide constants (status codes, choices, configurations)
- `mixins.py` - Reusable model/view mixins (timestamps, soft delete, etc.)
- `exceptions.py` - Custom exception classes for error handling

### requirements/
Python package dependencies organized by environment
- `base.txt` - Core packages required in all environments (Django, DRF, Celery, etc.)
- `development.txt` - Development-only packages (debugging tools, testing frameworks)
- `production.txt` - Production-only packages (gunicorn, monitoring tools)

### media/
User-uploaded files storage directory (images, videos, documents generated by users)

### static/
Static files for the project (CSS, JavaScript, images for Django admin/templates)

### staticfiles/
Collected static files for production deployment (generated by `collectstatic` command)
